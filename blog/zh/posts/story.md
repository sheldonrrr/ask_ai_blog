# Ask AI Plugin插件诞生是一个意外

> 关于这个插件的一个小故事。

2025年1月，我尝试把一本 Python 教程书读了两遍，但仍然难以入门。我甚至在一个专门的 Python 学习文件夹里塞满了一个又一个代码片段文件：我试图通过将更大、更宏观的模块分解成更小、更细粒度的学习和操作任务（基于每个代码片段文件）来建立我的 Python 代码基础。

当然，我可以悲观而清晰地预见一个月后会发生什么：即使文件夹中的代码文件越来越多，我知道一个月后，以我目前对不同模块的文本和认知工具，我仍然无法串联起足够完整的代码链来构建一个小产品。

后来，在整个过程中，我不断寻找新的工具，尤其是那些能在学习过程中提供更多透明度和基础知识的工具。

直到我重新发现了能够破解"代码链"的 AI 思维模式，我意识到当你不仅能看到"别人的"解决方案，还能看到他们在每一步回答问题时的思考过程，即使那个"别人"是一个 AI。

在"更透明、更具解释性的 AI 产品"的指引下，我理解并掌握了更多操作层面的常识，并写了一个非常小的产品叫 **Notifier**：利用系统推送通知，它使用一个包含单词和翻译的 JSON 表作为通知的标题和内容，在设定的时间随机发送以强化记忆。

完成的那一刻，我朝空气挥舞着拳头。

这是一次非常有趣的产品化尝试。假以时日，如果结合 Anki 的记忆概念，并从仅有 10 个单词和翻译的 JSON 数据集扩展，它可以成为一个独立的产品。

然后，AI IDE 产品爆发式流行。我惊讶地发现，虽然 AI 生成代码的方式、过程甚至结果都有些狂野，但如果你足够详细地描述你的需求（极其细粒度），你对代码的熟悉度会显著提高。后来，经过几个月的迭代，AI IDE 产品通过各种自动化工具，使得一个看似随机的 AI 越来越熟练，整个过程对于特定任务的透明度也提高了。

最终，Ask Grok 插件发布并迭代到稳定的 Ask AI Plugin 版本。

## 阅读不也是一样吗？

书籍就像一个迭代速度较慢的代码库，但它们有更严格的审核标准、专业的编辑和市场驱动的验证和筛选。

读者试图解开谜团，塑造自己的思维工具，并利用充足的数据构建自己独特的经验数据库。在这个塑造过程中，如何加速理解，如何将来自其他地方——也许是过去——的经验映射和转化到当前环境中，从自己独特的上下文信息中抽象和提炼新的智慧，这不也是人类"涌现智慧"的路径吗？

AI 无法提供所有正确的答案，但它可以提供加速解开谜团的可能性。欢迎使用 Ask AI Plugin 探索知识库中的任何谜团。

Ask AI Plugin 仍在持续更新，这是我的第一个开源项目，但不会是最后一个。
